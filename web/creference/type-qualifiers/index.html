Type Qualifiers

Author: Steve Russo
Last Updated: September 8, 2015

Type qualifiers are used to declare special properties of variables. In C, there are two type qualifiers, const and volatile. These may appear with any of the type specifiers when declaring variables.

const

The type qualifier const is used to place a variable in read-only memory. (These variables qualified as const are referred to as "constants.") In other words, a constant cannot have its value changed from whatever is initially assigned to it. For example: 

int main() {
	const int CONST_VAR = 0;
	// CONST_VAR = 123; will cause compiler error
	// error: read-only variable is not assignable
	return CONST_VAR;
}

The variable CONST_VAR will always have a value of 0. A program will not compile if there is an attempt to change the value of a constant anywhere in the code.

(Constants are typically named with all capital letters.)

volatile

The type qualifier volatile is used to make certain that no optimization is applied to a variable where it otherwise could be. This means that the variable's value is always the most recent value and not a cached one. An example of when this could be practically applied is when a multithreaded application has threads that share resources that are constantly being modified. By qualifying those resources as volatile, that is telling the compiler that no optimization should be applied so that the threads have the most recent values of each resource.

The syntax for declaring a variable as volatile is as follows. 

volatile int vi;
int volatile vi;

Both statements will declare a volatile integer variable named vi. The syntax for declaring a pointer to a volatile integer is the same. 

volatile int* volptr;
int volatile* volptr;

Although rarely ever used, the syntax for declaring volatile pointers to non-volatile data is as follows. 

int* volatile volptr;

And finally, almost as rarely used, the syntax for declaring volatile pointers to volatile data is as follows. 

int volatile* volatile volptr;
